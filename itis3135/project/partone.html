Client Project - Part One

Project Overview

  • The purpose of this website is to inform users what Charlotte Skyrise is. It also informs users about the sport of Ultimate Frisbee.
  • The intended users of this website are people interested in joining the team and learning about the sport. The primary target is college students already attending UNC Charlotte and students thinking about coming to UNC Charlotte.
  • The content included in this website is a home page, a page about the team, an interest form, a page about the sport, and a page of the roster and a photo gallery.

Page Design

Home Page

  • The name of the page is Skyrise Home.
  • The purpose of the page is to welcome the user and help them navigate the website.
  • The audience are people wanting to learn about skyrise, learn about ultimate frisbee, or people who are interested in joining the team.
  • This page will contain a team picture, a nav bar to get to the other pages, and a footer to get the teams social media.
  • This page will not require users to enter data.
  • This pages will not have any data fields.
  • This page will contain hyperlinks for the navbar and for the teams social media. There shouldn't be any drop downs.
  • This page will mostly involve clicking and scrolling. The user will interact with hyperlinks on the page in order to navigate the website.
  • The home page will mostly just be used for navigation and welcoming the user.

About the Team

  • The name of this page is About Skyrise.
  • The purpose of this page is to provide the user with information on the team.
  • The primary audience of this page are people who are trying to learn about the history of the team and what kind of team we are.
  • This page will contain the history of the team, statistics of the team, and also describe what kind of team we are.
  • This page will not require users to enter data.
  • This page will not have any data fields.
  • This page will contain hyperlinks for the navbar and the teams social media. It will also have links to learn about jacob fairfax as well as well as links to see scores from recent tournaments.
  • This page will mostly involve scrolling and clicking. The users will click on hyperlinks in order to learn about the teams history.
  • Jacob Fairfax is a player from skyrise who now plays professionally.

Interest Form

  • The name of this page is Skyrise Interest Form.
  • The purpose of this page is to give the team information on people interested in playing for skyrise.
  • The primary audience of this page are people who have seen what the team is about and now are interested in playing for the team.
  • This page will contain input fields that ask what position the user plays, how many years the user has played, and what their general knowledge of the sport is.
  • Yes this page will ask the users to enter data on the page.
  • This page will need to have data validation. For position played and general knowledge of the sport it should be a text input. For number of years played it should be a number input.
  • This page will maybe have a dropdown for positions or years played. It will also have a submit and reset button.
  • This page will involve typing and clicking on inputs. The users will also have to click on buttons to submit and reset the form.
  • This page will also probably will have the basic inputs that every page has such as name, email, etc.

About Ultimate Frisbee

  • The name of this page is About Ultimate Frisbee.
  • The purpose of this page is to inform the user about the sport and how it is played.
  • The primary audience of this page are people who have little to no experience with ultimate frisbee.
  • This page will contain information on vertical stack, how to play defense, what competitive play looks like, and how to self referee.
  • This page will not require users to enter data.
  • This page will not have not have any data fields.
  • This page will contain hyperlinks to pages that go more in depth on information that is on the page.
  • This page will mostly involve scrolling and clicking in order to navigate information.
  • This page will need to be organized very well because there is going to be a lot of information on it.

Roster

  • The name of this page is Skyrise Roster.
  • The purpose of this page is to show users the current players on the team.
  • The primary audience is going to be users trying to learn about the team.
  • This page will just contain a list of names, what year they are, and what position they play.
  • This page will not require users to enter data.
  • This page will not have any data fields.
  • This page will not contain any hyperlinks or drop downs.
  • This page will most likely just involve scrolling.

Dynamic Functionality

  • Headers and Footers of the page will be dynamically loaded so that changes to the header and footer have to only be done once.
  • One of the pages will include a form that users will fill out to tell the team how much experience they have. For example REI has a form on their page that allows users to give feedback.
  • One of the pages should change as you scroll. For example FPP as the user scrolls they walk down the aisle.
